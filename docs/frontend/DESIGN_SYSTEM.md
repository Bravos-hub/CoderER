# CodeER Command Center Design System

CodeER uses a dark incident-response visual language: navy operational surfaces, emergency red-orange actions, high-contrast typography, explicit state badges and monospace provenance data.

Components must be keyboard operable, use semantic landmarks, retain visible focus, avoid hover-only actions and provide loading, empty, error, disconnected and conflict states.

Responsive behavior collapses the desktop sidebar to icons on tablets and a bottom navigation bar on small screens. Dense evidence, patch and audit data remains horizontally or vertically scrollable without clipping controls.
